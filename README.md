# Arcana Flow — 유니버설 웨이트 커버플로우 타로

애플 뮤직의 **커버플로우**처럼 유니버설 웨이트(Rider–Waite–Smith, 1909 · 퍼블릭
도메인) 타로 78장을 넘겨 보고, **장 수를 고른 뒤 같은 페이지에서 바로 뽑아**
카드마다 긴 해석을 읽는 정적 웹앱. 외부 빌드 의존성 없는 순수 HTML/CSS/JS.

## 기능
- **커버플로우**: 78장 3D 플립(클릭·드래그·휠·방향키·슬라이더로 탐색).
- **12 테마**: 클래식 / 구미베어 / 아르누보 / 미드나잇 / 골드리프 / 파스텔 /
  네온 / 세피아 / 우드컷 / 사쿠라 / 오션 / 모노크롬. 카드 그림은 고정(RWS)이고
  테마는 배경·프레임·강조색·이미지 필터를 바꾸는 **CSS 스킨**이다. (저작권상
  구미베어/아르누보 등 12종 '실제 덱' 936장을 구할 수 없어, 테마는 비주얼
  스타일로 구현했다.)
- **시간(밀리초) 연동 추첨**: 버튼을 누른 그 순간의 `Date.now()` 밀리초 +
  `performance.now()` 마이크로초 + 암호학적 엔트로피를 섞어 시드를 만든다
  (`js/rng.js`). 같은 **시드 = 같은 결과**(재현 가능, 공유 가능). 정/역방향도
  같은 스트림에서 결정(역방향 38%).
- **아카이브**: 뽑은 스프레드를 `localStorage`에 시드와 함께 저장해 최신순
  리스트업·다시 보기·삭제(`js/archive.js`).
- **해석**: 카드마다 서사 → 상징 분석 → 빛과 그림자 → 정/역 직접 해석 →
  당신에게(절제된 바넘) → 더 깊은 거울 → 시간(과거·현재·미래) → **한 걸음**(행동
  처방) → 맺음의 구조적 해석. 엔진은 **좋은 리딩의 best practice 10가지**(카드
  우선·무중복·분량 강박 없음·방향 반영·빛/그림자 균형·절제된 바넘·주체성·행동
  가능·재현성·정직한 표기)를 강제해, 글자 수를 채우려 같은 말을 반복하던
  **중언부언을 제거**했다(`js/reading.js` 헤더 참조).

## 파일 구조
```
tarot/
  index.html
  css/styles.css      노션풍 UI · 커버플로우 3D · 12 테마 토큰
  js/rng.js           시간/밀리초 시드 PRNG + Fisher–Yates 추첨
  js/archive.js       localStorage 아카이브
  js/cards.js         78장 데이터셋(상징·서사 씨앗) + 손글 essay 오버라이드
  js/lexicon.js       엔진 공용 수사 라이브러리(바넘·깊이·원소 목소리…)
  js/reading.js       카드 1장을 장문 에세이(HTML)로 빚는 조립 엔진
  js/app.js           컨트롤러(테마·플로우·추첨·렌더·아카이브)
```

## 해석 콘텐츠 정책 — 손글 에세이 채우기 (다중 세션)
목표는 **78장 모두 손으로 쓴 3000단어급 에세이**다. 분량이 방대하므로 점진적으로
채운다.

- **지금**: 엔진(`reading.js`)이 카드별 씨앗 + 공용 수사 라이브러리를 조립해 모든
  78장에 대해 **중복 없는 해석(엔진 카드 ~450–600 어절)** 을 즉시 제공한다. 분량을
  목표로 채우지 않고 할 말이 끝나면 멈추므로, 손글 에세이로 교체할수록 더 깊어진다.
  메이저 0번 **The Fool**은 손글 풀 에세이(~1,300 어절)로 완성되어 있다.
- **이어서 채우는 법**: `js/cards.js`의 해당 카드 객체에 `essay: [ '<h3>…</h3>',
  '<p>…</p>', … ]` 배열을 추가하면, 엔진은 그 카드에 한해 조립을 건너뛰고 손글을
  그대로 렌더한다(`reading.js`의 `essayFromCard`가 `card.essay`를 우선 사용).
  결과 카드의 분량 배지에 `· 손글`이 표시된다.
- 권장 진행 순서: 메이저 아르카나 22장 → 마이너 56장.

## 배포 (GitHub Pages)
`.github/workflows/pages.yml`가 `tarot/` 디렉터리를 Pages 아티팩트로 올려 배포한다.
- 저장소 **Settings → Pages → Build and deployment → Source = GitHub Actions**로
  한 번 설정해 두면 된다(워크플로의 `configure-pages`가 권한이 있으면 자동
  활성화도 시도).
- `claude/tarot-card-webapp-oblwe` 또는 `main`의 `tarot/**` 변경 시, 또는 수동
  실행(`workflow_dispatch`)으로 배포된다.

## 로컬 실행
```bash
cd tarot && python3 -m http.server 8080   # → http://localhost:8080
```
